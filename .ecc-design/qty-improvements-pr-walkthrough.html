02 / 14
MOTIVATION

What was broken before this PR

🔴 Flat registry — one system at a time

Map<koqName, entry> only cached the active system. Switching required a full async reload.

🔴 Race conditions on rapid system changes

Two overlapping loadFormatAndParsingMapsForSystem() calls would corrupt caches.

🔴 No "ready" signal

Consumers had no reliable way to know when specs were available after init or provider changes.

🔴 Listener leak in FormatsProviderManager

Each formatsProvider = setter stacked a new listener without removing the old one.

🔴 Unsafe event mutation during emit

BeEvent uses an array — removing a listener mid-emit skips or double-fires subsequent listeners.

🔴 Consumers reinvent caching

DisplayUnitFormatter was forced to build its own Map<UnitSystem, Map<koq, SpecPair>> because the framework didn't provide it.

03 / 14
CORE CHANGE

Three-level spec registry

Before

Map<koqName, FormattingSpecEntry>
  • One entry per KoQ name
  • Only the active unit system cached
  • Same KoQ + different persistence unit = collision

After

Map<koqName, Map<persistenceUnit, Map<UnitSystemKey, FormattingSpecEntry>>>
  • All 4 unit systems pre-cached at init
  • Same KoQ, different persistence units coexist
  • System switching is now a synchronous map lookup

consumer impact   getSpecsByName() return type changed: FormattingSpecEntry | undefined → Map<string, FormattingSpecEntry> | undefined. Civil's KindOfQuantitySpecPair.tryCreate() calls this.

MIGRATION

getSpecsByName() → migration

Before (Civil code)

const entry = IModelApp .quantityFormatter .getSpecsByName(koqName); if (!entry) return undefined; return new KindOfQuantitySpecPair( koqName, entry.formatterSpec, // ← direct entry.parserSpec, // ← direct unitSystem );

After — two options

// Option A: use new getSpecsByNameAndUnit() const entry = IModelApp .quantityFormatter .getSpecsByNameAndUnit({ name: koqName, persistenceUnitName: "Units.M", system: unitSystem, // optional });
// Option B: adapt getSpecsByName() Map const map = IModelApp .quantityFormatter .getSpecsByName(koqName); const entry = map?.get("Units.M");

recommended Option A — getSpecsByNameAndUnit() — is the direct replacement. Clean, explicit, supports system pinning.

06 / 14
NEW IN CORE-BENTLEY

BeUnorderedEvent

Set-backed event where listeners can safely add/remove themselves during emission.

How it works

  • Uses a Set<Context> instead of an array
  • _emitDepth counter tracks nested emits
  • Removed listeners are tombstoned (set to undefined)
  • Tombstones cleaned up when outermost emit exits
  • Supports addOnce() for one-shot listeners

Why it's needed

  • FormatSpecHandle subscribes to reload events
  • Handles may be dispose()d during emission
  • BeEvent (array) would skip/double-fire
  • Civil already had a local copy — now canonical

NEW IN CORE-QUANTITY

FormatSpecHandle

Auto-refreshing, cacheable handle to a FormatterSpec + ParserSpec pair. Created via QuantityFormatter.getFormatSpecHandle().

Features

  • Keyed to KoQ + persistence unit + optional system
  • Subscribes to onFormattingReadyUnordered
  • Auto-refreshes specs on every reload
  • format(value) with toString() fallback
  • Implements Disposable — works with using
// Create const handle = IModelApp .quantityFormatter .getFormatSpecHandle( "CivilUnits.LENGTH", "Units.M", "metric" // optional pin ); // Use (sync, no await needed) const text = handle.format(1234.5); // Cleanup handle.dispose();

compare This is the upstream equivalent of Civil's KindOfQuantitySpecPair — same pattern, but managed by the framework.

08 / 14
LIFECYCLE

Two-phase ready flow

_reloadCore()
→
onBeforeFormattingReady
providers register async work
→
collector.awaitAll()
20s timeout, log warnings
→
onFormattingReady
consumers read specs

Phase 1 — Providers

onBeforeFormattingReady fires with a FormattingReadyCollector. Providers call collector.addPendingWork(promise) to register async loading (e.g., domain formats from schemas).

Phase 2 — Consumers

onFormattingReady / onFormattingReadyUnordered fire after all work settles. isReady is true. FormatSpecHandle auto-refreshes here.

Rejections are logged as warnings but never block the ready signal. whenInitialized (one-shot promise) resolves after the first successful init.

09 / 14
CONCURRENCY

enqueueReload — serialized, latest-wins

All reload paths now go through a single queue. Prevents overlapping async reloads from corrupting caches.

// Rapid unit system changes: only the last one runs await setActiveUnitSystem("metric"); // queued → runs await setActiveUnitSystem("imperial"); // queued → replaces pending await setActiveUnitSystem("usCustomary"); // queued → replaces again // Result: only "usCustomary" reload executes. // finalizeReload() fires once at the end.

Behavior on success

Drains queue → finalizeReload() → sets isReady = true → emits ready events. If a new reload was queued during finalize, it runs next.

Behavior on failure

Logs error. If _hasEverBeenReady, restores isReady = true so stale-but-usable specs remain accessible. Pending reloads still run.

INTERFACE CHANGE

FormatsProvider.getFormat()

// Before getFormat(name: string): Promise<FormatDefinition | undefined>
// After getFormat(name: string, system?: UnitSystemKey): Promise<FormatDefinition | undefined>

Why

  • Enables per-system format resolution at the provider level
  • Schema providers can return different composite formats for metric vs. imperial
  • QuantityFormatter now passes the target system when populating the three-level registry

civil impact   FormatSetFormatsProvider in ecschema-metadata and Civil's SchemaFormatsProvider already do per-system work internally. This change makes the interface match reality.

SMALLER FIXES

Two more things

impliedUnitSystem

New optional field on FormatsChangedArgs. When a provider fires onFormatsChanged with this set, QuantityFormatter auto-switches the active unit system to match.

this.onFormatsChanged.raiseEvent({ formatsChanged: "all", impliedUnitSystem: "metric", });

Listener leak fix

FormatsProviderManager now calls _removeProviderListener?.() before re-registering when the formatsProvider setter is called.

// Before: stacked listeners set formatsProvider(p) { this._provider = p; p.onFormatsChanged.addListener(…); } // After: clean swap set formatsProvider(p) { this._removeListener?.(); this._provider = p; this._removeListener = p.onFormatsChanged.addListener(…); }

NEW APIs

Readiness APIs on QuantityFormatter

isReady

Synchronous boolean. true after each successful reload, false during reloads.

whenInitialized

One-shot Promise<void>. Resolves after the first successful init, stays resolved forever.

onFormattingReady

Fires after every reload. Ordered (BeUiEvent). Use in UI components for rendering.

onFormattingReadyUnordered

Same signal, Set-backed (BeUnorderedUiEvent). Safe for handles that may dispose during emit.

onBeforeFormattingReady

Hook for code that supplies formats (e.g., FormatSetFormatsProvider, SchemaFormatsProvider). Fires before onFormattingReady with a FormattingReadyCollector so providers can call collector.addPendingWork(promise) to load domain formats asynchronously. The formatter waits for all registered work (20s timeout) before signaling ready.
